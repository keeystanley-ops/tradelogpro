## **DESIGN LANGUAGE AND THEME**

**Typography:** Use a modern, clean sans-serif font family such as Inter, system UI fonts, or similar that aligns with professional financial applications. Ensure consistent font weighting throughout the application—use medium weight for labels and headers, bold for key metrics and values, and normal weight for table content. The typography must feel cohesive and readable across all sections.

**Color Palette:** Maintain a light mode theme with white backgrounds for cards and tables, and a very light gray for the main page background to create subtle separation. Use semantic colors strictly for financial data: green for positive values, winning trades, and profit indicators; red for negative values, losing trades, and loss indicators. Use neutral grays for secondary text, borders, and inactive states. Use a primary brand color for interactive elements like buttons and links.

**Spacing and Layout:** Use a consistent spacing system throughout the application to ensure visual harmony. Margins and padding should feel breathable but compact enough to display data densely. Use responsive grid layouts for the metric cards and flexible table structures that adapt to content width. Avoid hardcoded dimensions; use relative sizing and flexbox or grid systems to ensure elements align proportionally.

**Visual Style:** Cards should have subtle borders and soft shadows to create depth. Inputs and dropdowns should have clean borders with rounded corners. Badges and status indicators should be pill-shaped or slightly rounded. The overall feel should be crisp, modern, and data-focused.

## **FUNCTIONAL REQUIREMENTS**

**Sorting:** All designated columns must support sorting by clicking the header. Toggle between ascending, descending, and no sort. Display visual indicators (arrows) showing the current sort direction.

**Filtering:** The Filters dropdown must open a panel allowing users to filter by symbol, status, date range, P&L range, and other criteria. Filters must apply immediately to the table and metric calculations.

**Row Selection:** Users must be able to select individual rows via checkboxes. The header checkbox must select or deselect all visible rows. When rows are selected, the Bulk actions button must become enabled.

**Bulk Actions:** The Bulk actions dropdown must offer options like Export, Delete, or Tag Selected. These actions must perform the intended operation on the selected rows.

**Account Switching:** The Account dropdown must allow switching between different accounts. Switching accounts must reload the trade data and recalculate all metrics for the selected account.

**Date Range:** The Date range dropdown must allow selecting predefined periods or a custom range. Changing the date range must filter the table and update all metric cards.

**Responsiveness:** The layout must adapt to different screen sizes. On smaller screens, metric cards should stack, and the table should support horizontal scrolling or card view conversion.
